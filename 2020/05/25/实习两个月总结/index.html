实习两个月总结

发表于 2020-05-25

编程思路上

  1. 写页面顺序：先分析页面的核心功能有哪些，想想用户的使用顺序是怎么样的
  2. 页面的书写：
    1. 表格的布局，数据的渲染显示
    2. 功能实现步骤：
      1. 确定参数，先定义好参数
      2. 初始化参数
      3. 拿到后台数据
        • 注意检查数据是不是为空，校验
        • 将数据渲染到页面上
        • 做完需要简单测试，参数是否正确传到了
      4. 注意：
        • url里一般不放参数，get请求不安全
        • 时间参数推荐用‘-’分隔，‘/’可能会和转义符冲突，注意参数格式前后一致
        • 注意去掉参数前后空格（trim方法）
        • 减少ajax的使用

工作常用的工具

miniUI

  1. 页面一加载就使用遮罩mini.mask({})，完成ajax就取消遮罩效果mini.unmask()

    查询设计器

后台

  1. 怎么把数据在aspx里拼接好，前台可以一次性渲染出来？（4.7）

    浏览器调试

  2. 可以使用 console.log() 、alert()方法在调试窗口上打印 JavaScript 值
  3. 打断点，查看出问题的在哪一行。在每个断点上，都会停止执行 JavaScript 代码，以便于我们检查 JavaScript 变量的值。
  4. debugger 关键字用于停止执行 JavaScript，并调用调试函数。这个关键字与在调试工具中设置断点的效果是一样的。如果没有调试可用，debugger 语句将无法工作。
    1. 开启 debugger ，代码在第三行前停止执行。
  5. 注释部分代码，看哪些代码可以正确运行，就说明这一小部分没问题。找其他地方的问题
  6. 注意nerwork中查看ajax请求的地址是否正确，返回的数据是否正确

    E-charts

  7. 如果父级容器的height/width属性设置为百分比的形式，那么echarts就会warning，且不能正常的生成图表。所以div容器的高度宽度必须指定为px

IE8的兼容问题

  1. 不支持nth-child特性。解决办法：使用:first-child+子代标签
